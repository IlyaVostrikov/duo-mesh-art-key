# DUO MESH — Видение проекта

## Идея

**DUO MESH** — это верифицированная экосистема для независимых художников, где каждый получает собственный виртуальный выставочный зал, а каждая работа сопровождается цифровым сертификатом подлинности (Art Key). Платформа решает три ключевые проблемы арт-рынка:

1. **Подлинность** — Art Key с SHA-256-хешем и цепочкой provenance-записей делает подделку бессмысленной. Без блокчейна, без газа, без кошельков — криптографическая защита, понятная обычному пользователю.

2. **Открытие художников** — вместо галерейного гейткипинга алгоритмическая и социальная дискавери: полнотекстовый поиск, курируемые подборки, взаимные подписки. Художник с нулевым бюджетом получает ту же инфраструктуру, что и галерея.

3. **Прямые продажи** — комиссия 12% против 30-50% традиционных галерей. Покупатель связывается с художником напрямую, платформа не держит деньги — сделка вне платформы на MVP, эквайринг в будущем.

Миссия: сделать рынок искусства прозрачным и доступным без инфраструктурного порога.

## Технический фундамент

### Методология: VIBE (github.com/di-sukharev/vibe)

Проект построен на VIBE-шаблоне — agent-first монорепозитории с Bun-воркспейсами. Ключевые архитектурные решения унаследованы от шаблона:

- **Монорепозиторий** (`bun workspaces`) — единый lock-файл, общие контракты, сквозной typecheck
- **Паттерн route → guard → service → DTO** — каждый слой изолирован и заменяем
- **Поверхностный сплит** — website (Astro SSG/SSR), webapp (React CSR), backend (Hono API) — разные потребители, разные стратегии рендеринга
- **Prisma + PostgreSQL 18** — типизированный доступ к данным с UUIDv7-ключами

### Стек

| Слой | Технология | Почему |
|---|---|---|
| Рантайм | Bun | Единый рантайм для скриптов, сервера и тестов |

| Деплой | DigitalOcean App Platform | Три app-сервиса из одного репо, авто-деплой из master |

### Модель данных (16 моделей)

```
User ─1:1─ Artist ─1:N─ Artwork ─1:1─ ArtKey
  │          │                  │
  │          │                  └─1:N─ ProvenanceRecord (цепочка владения)
  │          │                  └─1:N─ Inquiry (запросы на покупку)
  │          │
  │          └─1:1─ ExhibitionHall
  │          └─M:N─ Follow
  │
  └─1:1─ Collector ─M:N─ Collection ─M:N─ Artwork
  └─1:N─ Notification
```

Ключевые проектные решения схемы:
- **ArtKey.keyCode** = `DUO-YYYY-XXXXXXXX` — человекочитаемый идентификатор сертификата
- **ProvenanceRecord.prevRecordHash** = SHA-256 предыдущей записи — делает цепочку владения tamper-evident
- **GIN-индексы** на `artworks` для полнотекстового поиска по названию, описанию и имени художника
- **Статусная машина** artwork: DRAFT → LISTED → IN_EXHIBITION → SOLD/RESERVED

### Ролевая модель

| Роль | Доступ |
|---|---|
| Guest | Просмотр каталога, художников, залов, верификация Art Key |
| Artist | Создание/редактирование работ, настройка зала, просмотр статистики |
| Collector | Сохранение в избранное, подписки, отправка запросов на покупку |
| Admin | Модерация, верификация художников, управление featured-контентом |

## Принципы разработки

1. **Agent-first** — архитектура спроектирована так, чтобы AI-агент мог продуктивно работать: монорепозиторий, контракты, автогенерация типов, паттерн route→guard→service→DTO.

2. **SEO как продукт** — каждая страница artwork и художника индексируется. Schema.org-разметка на Astro-страницах. Art Key создаёт постоянный URL для верификации.

3. **Без блокчейна** — криптографическая защита (SHA-256, provenance-цепочка) даёт тот же уровень доверия, но без пользовательского трения. Токен на NFT можно добавить позже, но не как обязательство.

4. **Художник владеет аудиторией** — подписчики, контакты покупателей, история продаж — всё принадлежит художнику, не платформе. При уходе с платформы данные экспортируются.

5. **Русский + английский** — весь интерфейс и контент на двух языках. i18n на уровне роутов и контрактов.

DUO MESH занимает нишу между традиционными маркетплейсами (дорого, галерейный гейткипинг) и NFT-платформами (крипто-барьер, спекулятивный рынок). Мы — для живых художников с реальными работами.
