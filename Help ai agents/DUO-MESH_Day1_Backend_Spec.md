## Контекст

DUO MESH — платформа для независимых художников: у каждого виртуальный выставочный зал, у каждой работы цифровой сертификат подлинности (Art Key) с SHA-256 и provenance-цепочкой. Без блокчейна.

**Цель этого дня:** наполненная двуязычная (RU+EN) база, где публичный каталог выглядит живым, есть несколько 3D-работ, и каждая работа имеет валидный Art Key с provenance-цепочкой. Это фундамент под демо-презентацию. Загрузка файлов пользователями в эту неделю **не делается** — ассеты засеиваются напрямую.

**Стек (не менять):** Bun workspaces (монорепо), Hono + Zod 4 (API), PostgreSQL 18 + Prisma (UUIDv7-ключи, GIN-индексы для полнотекстового поиска), Astro 6 (website), React 19 + Vite + TanStack Router (webapp), DigitalOcean Spaces (S3-совместимое хранилище).

## Задача 1 — Дельта схемы под 3D

Добавить в модель `Artwork` различение медиума и поля для 3D. Сначала прочитать актуальный `schema.prisma`, затем привести к следующей семантике (имена сохраняй, если совпадают; адаптируй типы под реальную схему):

```prisma
enum MediaType {
  IMAGE_2D
  MODEL_3D
}

enum CreationSoftware {
  BLENDER
  ZBRUSH
  SCAN          // фотограмметрия / 3D-скан
  OTHER
}

model Artwork {
  // ... существующие поля ...
  mediaType   MediaType        @default(IMAGE_2D)
  posterUrl   String           // обязателен ВСЕГДА: обложка для гридов, SEO, OG-карточек
  modelUrl    String?          // GLB, обязателен при mediaType = MODEL_3D
  software    CreationSoftware?
  isScanned   Boolean          @default(false)
  polyCount   Int?
  // ...
}
```

Правила валидации (на уровне Zod-контракта и/или сервиса):
- `posterUrl` непустой для любой работы.
- если `mediaType = MODEL_3D` → `modelUrl` обязателен и указывает на `.glb`.
- если `mediaType = IMAGE_2D` → `modelUrl` = null.

Миграция: `bunx prisma migrate dev --name add_3d_media`. GIN-индексы для полнотекстового поиска уже существуют (MVP) — не пересоздавать, только убедиться, что сидируемые текстовые поля (название, описание, имя художника) заполнены на RU и EN, чтобы демо поиска работало.

## Задача 3 — Сид-данные

Скрипт `prisma/seed.ts` (или существующий сид-механизм). Генерируй детерминированно и идемпотентно (повторный запуск не плодит дубликаты — upsert по стабильному slug/keyCode).

**Ростер: 6 художников.** Большинство 2D для визуальной полноты каталога; минимум один полностью 3D-нативный — его зал станет ключевым «вау»-моментом демо.

| # | Медиум | Стиль | Работ | Примечание |
|---|---|---|---|---|
| 1 | 2D painting | abstract | 6 | |
| 2 | 2D digital | surreal | 5 | |
| 3 | 2D photography | monochrome | 5 | |
| 4 | 2D mixed/print | contemporary | 4 | |
| 5 | **3D sculpture (GLB)** | figurative / scan | 4 | **3D-нативный зал — витрина демо** |
| 6 | 2D + 1–2 × 3D | new media | 5 | смешанный, демонстрирует оба медиума в одном зале |

Итого ≈ 29 работ, из них ≈ 6 — 3D (`MODEL_3D`).

Для каждого художника: `User` + `Artist` (bio и artistStatement на **RU и EN**, соцсети) + `ExhibitionHall` (обложка, описание). Для каждой работы: название и описание на RU+EN, цена (диапазоны $200–$8000, разные), техника/размеры, `mediaType`, `posterUrl`, при 3D — `modelUrl`/`software`/`isScanned`/`polyCount`. Статусы работ варьировать (`LISTED`, часть `IN_EXHIBITION`, 2–3 `SOLD` — чтобы provenance-цепочки имели больше одной записи).

### Генерация Art Key (на каждую работу)

- `keyCode` формата `DUO-{YYYY}-{XXXXXXXX}`, где `YYYY` — год выдачи, `XXXXXXXX` — 8 символов в верхнем регистре (hex или base32 от случайного источника). Пример: `DUO-2026-7F3A9C2E`. Уникальность гарантировать.
- `integrityHash` = `sha256(canonicalJSON({ artworkId, keyCode, artistId, issuedAt }))`, hex. **canonicalJSON** = ключи отсортированы лексикографически, без пробелов. Зафиксируй эту функцию и переиспользуй везде.

### Provenance-цепочка (tamper-evident)

Записи `ProvenanceRecord` упорядочены полем `sequence` (0,1,2…). Хеш-цепочка строится так:

- **Genesis (sequence = 0, eventType = `CREATED`)**: `prevRecordHash` = `integrityHash` Art Key (якорит цепочку к сертификату).
- **recordHash каждой записи** = `sha256(canonicalJSON({ artworkId, sequence, eventType, actor, occurredAt, prevRecordHash }))`.
- **prevRecordHash следующей записи** = `recordHash` предыдущей.

Если поля `recordHash` нет в схеме — добавить (String) и реконсилировать. Для сида: у каждой работы genesis-запись `CREATED`; у `IN_EXHIBITION` добавить `LISTED`/`EXHIBITED`; у `SOLD` добавить `SOLD` с фиктивным коллекционером. Цепочка обязана проходить проверку пересчётом.

## Задача 4 — Спайк просмотрщика 3D (де-риск дней 2–3)

В webapp подключить `@google/model-viewer` (npm или CDN) и собрать одну тестовую страницу/роут с минимальной интеграцией на одной сидовой GLB:

```html
<model-viewer
  src="{modelUrl}"
  poster="{posterUrl}"
  camera-controls
  auto-rotate
  ar
  shadow-intensity="1"
  loading="lazy">
</model-viewer>
```

Проверить: модель грузится, poster показывается до загрузки, орбита/зум работают. Цель — снять технический риск до того, как дизайн-трек начнёт верстать зал.
