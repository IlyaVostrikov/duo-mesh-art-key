# DUO MESH — День 1 · Спецификация для агента-дизайнера

> **Инструкция агенту.** Это исполняемая дизайн-спецификация для галереи нового поколения с упором на 3D. Значения ниже — конкретные и обязательные к реализации (CSS-переменные, type scale, поведение компонентов). Два решения помечены как «подтверждаемые» — реализуй с указанным дефолтом, но они спроектированы как легко заменяемые токены. Не вводи генерик-маркетплейс-эстетику (анти-референс — Etsy/Saatchi-грид).

## Контекст и север

DUO MESH — онлайн-выставка нового поколения, не маркетплейс. Редакционно, пространственно, дорого. Главный визуальный аргумент — **3D-работы** (скан или Blender/ZBrush, доставляются как GLB). Весь язык работает на то, чтобы работа «дышала» на экране. Цель дня — задать визуальную основу, на которой дни 2–3 соберут выставочный зал. Контент двуязычный (RU+EN) — **все шрифты обязаны иметь кириллицу.**

## Принципы

1. **Редакция, не витрина** — крупная типографика, асимметрия, воздух вместо плотной сетки.
2. **Пространство как роскошь** — щедрые отступы; работа всегда важнее хрома интерфейса.
3. **3D — герой** — медиум заявляет о себе ещё до клика; интерактив на детальной странице и в зале.
4. **Движение дисциплинированно** — тонко и предсказуемо; всё ломкое отключается на проекторе и при `prefers-reduced-motion`.
5. **Хром тихий, цвет несут работы** — UI хроматически сдержан, чтобы не конкурировать с искусством.

## Решение №1 (подтверждаемое): «чёрный бокс»

Направление по умолчанию — **тёмная галерея (black box)**: тёмный фон заставляет 3D-рендеры и подсвеченные работы выглядеть драматичнее и читается как современно/премиально. Это сильнейший сигнал «новой школы». Альтернатива — «белый куб» (музейная строгость): реализуется простой заменой нейтральных токенов. По умолчанию строим black box.

## Дизайн-токены (CSS-переменные)

```css
:root {
  /* Нейтрали — black box */
  --bg:            #0B0B0D;   /* фон-канва, не чистый чёрный */
  --surface:       #141417;   /* приподнятые поверхности, карточки */
  --surface-2:     #1C1C21;   /* ховер/вложенные поверхности */
  --border:        #2A2A31;
  --text:          #F5F5F4;   /* основной */
  --text-secondary:#A1A1AA;
  --text-muted:    #71717A;

  /* Решение №2 (подтверждаемое): акцент.
     Используется ТОЛЬКО для аффордансов — ссылки, бейдж 3D/AR,
     фокус-кольца, активные состояния. Не заливать им крупные плоскости. */
  --accent:        #C6FF3A;   /* электрический lime — «new school»; swappable */
  --accent-ink:    #0B0B0D;   /* текст на акценте */

  /* Радиусы — минимальные, редакционная резкость */
  --radius-sm: 2px;
  --radius:    4px;
  --radius-lg: 6px;

  /* Тени для тёмного UI — тонкие, через свет, не «коробку» */
  --elev-1: 0 1px 2px rgba(0,0,0,.4);
  --elev-2: 0 8px 30px rgba(0,0,0,.5);

  /* Движение */
  --dur-fast: 150ms;
  --dur:      250ms;
  --dur-slow: 400ms;
  --ease:     cubic-bezier(.2,.0,.0,1);
}
```

Спейсинг — 4px-база: `4 · 8 · 12 · 16 · 24 · 32 · 48 · 64 · 96 · 128`. Контейнер контента до ~1280px; зал может выходить шире для hero.

## Типографика

Все шрифты — с кириллицей (RU+EN). Загружать через self-host или Google Fonts с латиницей+кириллицей.

- **Display** (hero, имена художников, крупные заголовки работ): `Unbounded` — вариативный, выразительный, полная кириллица. Использовать **скупо**, только для крупнейших акцентов.
- **Текст / UI** (всё остальное, включая секционные заголовки): `Inter` — полная кириллица, отлично читается в мелком кегле.
- Опциональная редакционная альтернатива для statement/цитат, если захотят «музейности»: `Cormorant` (high-contrast serif, кириллица).

Type scale (rem, базовый 16px): `0.75 · 0.875 · 1 · 1.25 · 1.5 · 2 · 3 · 4.5 · 6`. Hero-display — крупный (`4.5–6rem` на десктопе), плотный трекинг. Body — `1rem`, line-height ~1.6. Лейблы/мета — `0.75–0.875rem`, увеличенный трекинг, верхний регистр для меток-категорий.

## Компонент: карточка работы (и чем отличается 3D)

Базовая 2D-карточка: `posterUrl` (4:5 или авто), под ней — название (display, малый кегль), имя художника (`--text-secondary`), цена/статус. Ховер: лёгкое поднятие `--surface → --surface-2`, появление мета-строки. Состояния: skeleton при загрузке, понятный empty.

**3D-карточка отличается явно:**
- бейдж в углу — `3D` (и `AR`, если применимо), залит `--accent`, текст `--accent-ink`;
- при ховере — медленный авто-rotate превью (или короткая зацикленная анимация poster→3D), отключается при `prefers-reduced-motion`;
- в гриде показывается **только poster** (производительность и SSG); интерактивный `<model-viewer>` грузится лишь на детальной странице и в зале.

## Спецификации экранов (wireframe-уровень, точно для реализации)

### 1. Каталог / индекс галереи
Редакционный, не плотный грид. Регионы: компактный хедер (лого слева, навигация, поиск, вход); опциональная hero-строка с одной featured-работой/подборкой; далее адаптивная сетка карточек (десктоп 3–4 колонки, планшет 2, мобайл 1) с щедрыми гаттерами. Фильтры — ненавязчивая боковая/верхняя панель (стиль, медиум `2D/3D`, цена). 3D-карточки перемешаны с 2D, но визуально маркированы бейджем.

### 2. Выставочный зал (главная витрина)
Регионы сверху вниз: **hero/обложка** зала (большое изображение или фон, имя художника в `Unbounded`); **artist statement** на видном месте (можно `Cormorant`), переключатель RU/EN; **featured-работа крупно** (для 3D — встроенный интерактивный просмотрщик прямо в hero); далее **курируемая раскладка работ** с воздухом и намеренной асимметрией (не равномерный грид) — чередование крупных и мелких. Адаптив: на мобайле всё в одну колонку, hero сохраняет масштаб.

### 3. Детальная страница работы (с 3D)
Двухзонный лейаут на десктопе: слева — медиа (для 2D крупный `posterUrl`; для 3D — `<model-viewer>` во всю зону), справа — название (display), художник (ссылка в зал), цена/статус, описание (RU/EN), техника/размеры, и блок **Art Key**: `keyCode`, видимый SHA-256, ссылка на верификацию, развёрнутая provenance-цепочка. CTA «Связаться с художником». Мобайл — стек: медиа сверху, информация ниже.

## Правила презентации 3D и атрибуты `<model-viewer>`

В гридах — `posterUrl`. Интерактив — только detail/зал. Базовые атрибуты:

```html
<model-viewer
  src="{modelUrl}"
  poster="{posterUrl}"
  camera-controls
  auto-rotate                 
  rotation-per-second="20deg" 
  ar ar-modes="webxr scene-viewer quick-look"
  shadow-intensity="1"
  exposure="1"
  environment-image="neutral"
  loading="lazy"
  reveal="auto">
</model-viewer>
```

Гард-рейлы производительности: ленивая загрузка только когда просмотрщик во вьюпорте; авто-rotate медленный; poster обязателен для мгновенного first paint; на мобайле не запускать тяжёлые модели в гридах никогда.

## Дисциплина движения / безопасность на проекторе

Только тонкие входные fade/translate; auto-rotate — медленный. Никаких параллаксов-каруселей, которые лагают на проекторе. Полное уважение `prefers-reduced-motion` (всё нелинейное отключается).
